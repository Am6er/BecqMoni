## Согласование SNR RJMCMC ↔ FWHMPeakFinder (сделано в боевом коде)

Рассогласование: у finder SNR — matched‑filter z‑оценка `signal/√(Σ d·k²)` (PeakFilter.convolve,
нуль‑суммовое ядро), а у RJMCMC `candidate.Snr = √ΔD` — корень из χ²‑статистики с ~2 ст. свободы
(центр+амплитуда, с реоптимизацией фона). Обе сравнивались с одним `Min_SNR` в
`AppendRjmcmcPeaks`, причём ΔD‑гейт заранее требовал `(1.5·Min_SNR)²` (ExtraSnrMultiplier=1.5),
т.е. extra‑пики держали планку в 1.5 раза выше finder'а.

Исправления (`RjmcmcPeakDeconvolver.cs`, `RjmcmcConfig.cs`):

1. `candidate.Snr` теперь = `ResidualSnr` — matched‑filter z по остатку против reduced‑модели,
   `Σ r·p / √(Σ max(1,λ)·p²)` — та же математика, что у finder. Сравнение с `Min_SNR` стало
   «яблоки с яблоками».
2. `ExtraSnrMultiplier` 1.5 → 1.0 — одинаковый порог обнаружения для finder и деконволюции.
3. ΔD‑гейты получили поправку Уилкса на 2 ст. свободы: `TargetSnr² + 2`.

На данных recall‑нейтрально (гейты не были лимитирующими: у принятых кандидатов residSNR 17–440),
но пороги теперь статистически честные — «SNR=6» у extra‑пика означает то же, что у finder‑пика.

## Что мешает Ra‑226 (диагностика)

- 1730/1764 слиты в один пик p1747 (разнос 34 кэВ < 0.6 FWHM); 2118/2204 → p2195;
  1402/1408/1378 → p1383. Sparrow‑предел, анкер фиксирован в центроиде бленда.
- Систематический сдвиг калибровки −3…−9 кэВ на высоких энергиях: 934→p928 (d6.1!),
  K‑40 1461→p1452 (d8.8) — промахи на волоске от допуска. Лечится рекалибровкой спектра,
  а не деконволюцией.

## Проверенные тупики (не повторять)

- `samples=4000, burnin=1000` — **ухудшает** Th (15→13): длинные цепи размывают occupancy
  плечевых кандидатов. Дефолт 1500/500 оптимален.
- Ослабление occ 0.7→0.6, corr 0.55→0.45, closeAnchor 0.6→0.5 — новых табличных линий нет,
  только ложные низкоэнергетические extra (62/66/73/229 кэВ). Гейты возвращены к дефолтам.
- `--snr=3` на Ra — finder и деконв новых линий не дают.

## Литература: куда усиливать RJMCMC дальше

1. **Split/merge‑ходы** (Bayesian nonparametric peak identification, arXiv:1107.5407) — замена
   «анкер + робкий сосед» на честное расщепление бленда: анкер 1747 → пара 1730+1764. Требует
   освободить позицию анкера в состоянии сэмплера. Самый перспективный шаг для Ra.
2. **Replica exchange + простые RJ‑ходы** (Fast Bayesian Deconvolution, arXiv:2011.13301) —
   обмен между темперированными репликами вместо длинных цепей; лечит залипание в моде
   «всё поглотил фон».
3. **Delayed rejection RJMCMC** (Green & Mira 2001) — второй шанс отвергнутым birth‑ходам
   с меньшей амплитудой; дёшево встраивается в ProposeBirth.
4. **Data‑driven birth уже есть** (residual‑weighted DrawBirthChannel) — соответствует
   рекомендациям, менять не нужно.

## Рекомендованная конфигурация

`--snr=6 --use-deconvolution=true --roi-radius=4 --max-extra=4` для Th‑подобных;
для Ra достаточно `--snr=4` (деконв нейтральна). Occupancy/корреляция — дефолтные 0.7/0.55.
